Paperless Generator Portal

Self-hosted Web-App, die den Paperless-ngx Setup Generator dauerhaft bereitstellt — mit Login, Einstellungsmenü und automatischem API-Proxy. Deployment in einem Befehl auf einem Proxmox-Host.

Überblick

Der Container läuft ein kleines Flask-Backend, das den unveränderten Generator ausliefert, ein Login-Gate davorsetzt und die Paperless-API über einen Proxy anspricht:

Browser ──▶ Flask (Port 8080)
             ├─ /login, /logout   Login-Gate (Standard admin/admin)
             ├─ /settings         Paperless-URL + Token, Passwort ändern
             ├─ /                 Generator (unverändert) + Vorkonfig-Zeile
             └─ /api/*  ──▶ Proxy ──▶ Paperless (Token wird eingespritzt)

Warum kein CORS mehr? Der Generator ruft /api/… same-origin auf; das Backend leitet an das konfigurierte Paperless weiter und spritzt den gespeicherten Token ein. Dadurch entfällt jede PAPERLESS_CORS_ALLOWED_HOSTS-Konfiguration in Paperless.

1. Schnellstart (Ein-Zeiler)

Auf der Proxmox-Host-Shell (nicht im Container) ausführen:

bash -c "$(curl -fsSL https://raw.githubusercontent.com/martin86x/paperless-generator-portal/main/proxmox-install.sh)"

Das Skript fragt ein paar Werte ab (Container-ID, Storage, Bridge, Disk, RAM) — die Defaults passen für den Heimgebrauch. Danach läuft es automatisch durch und hakt jeden Schritt grün ab:

 ✓ Proxmox-Host erkannt
 ✓ Template bereit (debian-12-standard)
 ✓ LXC 200 erstellt (privileged, Nesting, onboot)
 ✓ Container gestartet, Netzwerk erreichbar (IP 192.168.1.x)
 ✓ Docker installiert, Projekt geholt & Container gebaut

── Selbstcheck ──
 ✓ Container läuft (docker ps)
 ✓ Healthcheck: healthy
 ✓ HTTP 200 auf http://192.168.1.x:8080/login
 ✓ Autostart aktiv (onboot=1, restart=unless-stopped)

Voraussetzung: Das Skript legt einen privileged LXC mit Nesting an (nötig, damit Docker im Container läuft). Das erledigt es selbst — es sind keine Vorarbeiten am LXC nötig.

2. Login & Passwort ändern

Nach dem Durchlauf zeigt das Skript die URL. Im Browser öffnen:

FeldWert
URLhttp://<lxc-ip>:8080
Benutzeradmin
Passwortadmin

Wichtig: Nach dem ersten Login unter Einstellungen → Passwort ändern sofort ein eigenes Passwort setzen.

3. Paperless-Verbindung einrichten

In den Einstellungen die Verbindung eintragen und speichern:

FeldBeispielHinweis
Paperless-URLhttp://192.168.1.200:8000Basis-URL inkl. Port, ohne /api
API-Tokendein Tokensiehe unten

Token holen: In Paperless anmelden → Einstellungen → Mein Profil → API-Token (bzw. unter /api/token/). Der Token wird serverseitig gespeichert und vom Proxy automatisch bei jedem API-Aufruf eingespritzt — im Generator selbst musst du nichts eintragen.

4. Generator nutzen

Oben im Menü auf Generator klicken. Er ist bereits mit deiner Paperless-Instanz verbunden (keine Eingabe in der Verbindungsmaske nötig) — alle Werkzeuge wie Health-Check, Matching-Audit oder der Direkt-Modus funktionieren sofort.

5. Generator aktualisieren

Kommt eine neue Generator-Version, im Container das Repo aktualisieren und neu bauen:

cd /opt/paperless-generator-portal
git pull
docker compose up -d --build

Auf dem Entwicklungsrechner übernimmt update.ps1 das neueste dist/index.html aus dem Generator-Projekt in site/.

6. Manueller Betrieb (ohne Proxmox-Skript)

Läuft auf jedem Docker-Host. Im Projektordner:

docker compose up -d --build

Erreichbar unter http://localhost:8080. Die Konfiguration liegt im Volume ./config/config.json.

7. Sicherheit

  • Der Paperless-Token wird im config-Volume gespeichert (Klartext), damit der Proxy ihn einspritzen kann. Gedacht für den Betrieb im eigenen Heimnetz.
  • Das Admin-Passwort wird nur als Hash gespeichert; das Standard-Passwort admin unbedingt ändern.
  • Alle Routen außer /login und dem Healthcheck sind login-pflichtig.

8. Optionale Homelab-Integration

gethomepage-Kachel

- Paperless Generator:
    href: http://192.168.1.x:8080
    description: Setup Generator Portal
    siteMonitor: http://192.168.1.x:8080/healthz

Uptime Kuma

Monitor vom Typ HTTP(s) auf http://<lxc-ip>:8080/healthz (erwartet Status 200) — meldet, wenn der Container ausfällt.

Portainer

Der Container erscheint als paperless-generator-portal und lässt sich dort verwalten (Logs, Neustart, Update).
