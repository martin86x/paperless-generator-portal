❓ Wie funktionieren Profile? (einmal lesen)

Ein Profil = eine Paperless-Instanz. Es bündelt zwei Dinge: die Verbindung (URL + API-Token) und die komplette Generator-Konfiguration (Tags, Dokumenttypen, Felder, Korrespondenten, Speicherpfade, Workflows). So verwaltest du mehrere Instanzen — z. B. Haushalt, Firma, Testsystem — sauber getrennt.

Das „aktive" Profil steuert alles. Genau ein Profil ist aktiv (grünes ● aktiv). Es bestimmt:

  • womit der Generator arbeitet (er lädt die Config des aktiven Profils),
  • an welche Instanz der Proxy alle /api/-Aufrufe weiterleitet.

Mit ▶ Aktivieren wechselst du und landest direkt im Generator mit den Daten dieses Profils.

⚠ Wichtig — Token gehört hierher, nicht in den Generator.
Der Generator hat eine eigene Sektion 01 „Verbindung". Im Portal-Betrieb ist die wirkungslos: der Proxy verwirft einen dort eingetippten Token und spritzt den aus dem Profil ein. Den API-Token also immer hier beim Profil unter „Verbindung" eintragen. (Steht im Token-Feld eine URL statt eines Tokens → roter Status / HTTP 401.)

Die Aktionen je Profil (unter „Verbindung · Umbenennen · Löschen"):

  • Verbindung — URL + API-Token setzen. Leeres Token-Feld = bestehendes behalten. Der Status oben zeigt live grün/rot.
  • Umbenennen — nur die Anzeige, ändert nichts an den Daten.
  • Produktivsystem — blendet überall einen auffälligen Warnbalken ein, damit du nie versehentlich die Live-Instanz bearbeitest.
  • Nur lesen — der Proxy blockt jeden Schreibzugriff (POST/PUT/PATCH/DELETE) auf diese Instanz. Härtester Schutz.
  • Löschen — entfernt das Profil samt gespeicherter Config (Dokumente in Paperless bleiben unberührt).

Speichern & Historie. Im Generator speicherst du Änderungen mit „💾 Profil speichern". Der vorige Stand landet automatisch in der Versionshistorie — jederzeit per „↩ Wiederherstellen" zurückholbar. Ein neues, leeres Profil hat noch keine Config → der Generator startet mit seinen Standard-Daten.

Sichern. „⬇ Alle Profile sichern" lädt eine JSON mit allen Profilen (Tokens entschlüsselt → auf anderem Host einspielbar). „⬆ Wiederherstellen" ersetzt alle Profile (vorher wird automatisch gesichert).
